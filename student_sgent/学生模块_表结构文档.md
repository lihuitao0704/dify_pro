# 学生智能助手模块 — 数据库表结构文档

> **数据库**：`hambaki_3`  
> **字符集**：utf8mb4  
> **引擎**：InnoDB  
> **关联**：所有表通过 `student_id` 与 `test.students.id` 做业务关联  
> **设计原则**：不使用物理外键，业务关联在 Service 层做显式 JOIN 查询  
> **时间字段**：统一使用 MySQL `CURRENT_TIMESTAMP`（数据库侧生成，避免多服务器时区不一致）

## 1. conversation_sessions — 学生会话主表

**说明**：记录学生会话的元信息，每个学生可有多条会话记录。通过 `session_id`（VARCHAR）与 `conversation_messages` 做业务关联。

| 字段名 | 类型 | 是否NULL | 默认值 | 说明 |
|--------|------|----------|--------|------|
| `id` | BIGINT | 否 | 自增 | 主键 |
| `session_id` | VARCHAR(64) | 否 | — | 会话唯一标识（UNIQUE） |
| `student_id` | BIGINT | 否 | — | 学生ID（必填，仅记录学生） |
| `status` | ENUM | 否 | `active` | 会话状态：`active` / `closed` / `timeout` |
| `last_message_time` | DATETIME | 是 | NULL | 最后消息时间 |
| `message_count` | INT | 否 | 0 | 消息总数（冗余计数器，避免频繁 COUNT） |
| `close_time` | DATETIME | 是 | NULL | 会话关闭时间 |
| `create_time` | DATETIME | 否 | CURRENT_TIMESTAMP | 创建时间 |

**索引**：`session_id`(UNIQUE) · `student_id` · `status`

---

## 2. conversation_messages — 消息明细表

**说明**：记录每条会话中的具体消息，包含 AI 意图识别和情绪分析结果。情绪字段是心理预警自动触发的数据来源。  
**触发规则**：`emotion_score < 30` 或 `trigger_keywords` 含高危词 → 自动创建 `risk_interventions` 记录

| 字段名 | 类型 | 是否NULL | 默认值 | 说明 |
|--------|------|----------|--------|------|
| `id` | BIGINT | 否 | 自增 | 主键 |
| `session_id` | VARCHAR(64) | 否 | — | 关联会话ID |
| `role` | ENUM | 否 | — | 消息角色：`user` / `assistant` / `system` |
| `content` | TEXT | 否 | — | 消息内容 |
| `intent` | VARCHAR(64) | 是 | NULL | AI识别意图 |
| `emotion_tag` | VARCHAR(64) | 是 | NULL | 情绪标签（焦虑/平稳/低落/愤怒等） |
| `emotion_score` | INT | 是 | NULL | 情绪分值 0-100，越高越积极 |
| `trigger_keywords` | JSON | 是 | NULL | AI提取触发关键词（JSON数组） |
| `tokens_used` | INT | 是 | NULL | 消耗Token数 |
| `response_time_ms` | INT | 是 | NULL | 响应耗时（毫秒） |
| `create_time` | DATETIME | 否 | CURRENT_TIMESTAMP | 创建时间 |

**索引**：`session_id` · `intent` · `emotion_tag` · `create_time`  
**约束**：`emotion_score >= 0 AND emotion_score <= 100`

## 3. emotion_profile_snapshots — 心理画像表

**说明**：记录学生当前心理状态，`student_id` 唯一（一人一条快照）。每次情绪交互后更新。历史明细数据存在 `conversation_messages.emotion_*` 字段中。

| 字段名 | 类型 | 是否NULL | 默认值 | 说明 |
|--------|------|----------|--------|------|
| `id` | BIGINT | 否 | 自增 | 主键 |
| `student_id` | BIGINT | 否 | — | 学生ID（UNIQUE，一人一条） |
| `latest_emotion_tag` | VARCHAR(64) | 是 | NULL | 最新情绪标签 |
| `emotion_score` | INT | 是 | NULL | 情绪分值 0-100 |
| `risk_level` | ENUM | 否 | `low` | 风险等级：`low` / `medium` / `high` |
| `emotion_history` | JSON | 是 | NULL | 历史情绪波动数据（JSON数组） |
| `last_interaction_time` | DATETIME | 是 | NULL | 最近一次交互时间 |
| `update_time` | DATETIME | 否 | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |
| `create_time` | DATETIME | 否 | CURRENT_TIMESTAMP | 创建时间 |

**索引**：`student_id`(UNIQUE) · `risk_level`

| `detail` | TEXT | 是 | NULL | 详细描述 |
| `status` | ENUM | 否 | `pending` | 处理状态：`pending` / `processing` / `resolved` / `closed` |
| `priority` | ENUM | 否 | `medium` | 优先级：`low` / `medium` / `high` / `urgent` |

## 7. deadline_reminders — 考务提醒表

**说明**：记录学生关键学业节点（论文DDL、考试时间、申请截止、签证到期等）。支持通用提醒（`student_id=NULL`）和个人提醒。提醒粒度为**天数级**，适合中长期的截止日期预警。

**查询示例（未来7天到期）**：
```sql
SELECT * FROM deadline_reminders
WHERE deadline BETWEEN NOW() AND DATE_ADD(NOW(), INTERVAL 7 DAY)
  AND status = 'pending'
ORDER BY deadline ASC;
```

| 字段名 | 类型 | 是否NULL | 默认值 | 说明 |
|--------|------|----------|--------|------|
| `id` | BIGINT | 否 | 自增 | 主键 |

| `title` | VARCHAR(255) | 否 | — | 节点名称 |
| `description` | TEXT | 是 | NULL | 描述 |
| `deadline` | DATETIME | 否 | — | 截止时间 |
| `reminder_days` | JSON | 是 | NULL | 提前提醒天数配置，如 `[7, 3, 1]` |

## 全局设计说明

### 时间字段规范

| 字段 | 机制 | 说明 |
|------|------|------|
| `create_time` | `DEFAULT CURRENT_TIMESTAMP` | INSERT 时由 MySQL 自动填充 |
| `update_time` | `DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | INSERT 和 UPDATE 时由 MySQL 自动填充 |

**为什么用数据库侧时间而非 Python 侧**：避免多服务器多时区场景下的时间不一致。

### 索引命名规范

格式：`idx_{表简称}_{字段名}`

| 表 | 表简称 |
|----|--------|
| `conversation_sessions` | `session` |
| `conversation_messages` | `msg` |
| `emotion_profile_snapshots` | `profile` |
| `risk_interventions` | `intervention` |
| `feedback_tickets` | `ticket` |
| `academic_schedules` | `schedule` |
| `deadline_reminders` | `deadline` |
| `study_intentions` | `intention` |
| `student_applications` | `application` |

### CHECK 约束

| 表 | 约束 | 说明 |
|----|------|------|
| `conversation_messages` | `emotion_score >= 0 AND <= 100` | 情绪分值范围 |
| `feedback_tickets` | `satisfaction >= 1 AND <= 5` | 满意度范围 |

> **注意**：CHECK 约束要求 MySQL 8.0.16+。NULL 值不计入 CHECK 校验，允许在可选字段中插入 NULL。

### 关联关系

| 关联 | 方式 |
|------|------|
| `conversation_sessions` ← `conversation_messages` | `session_id` (VARCHAR) |
| `conversation_messages` → `risk_interventions` | `source_message_id` (BIGINT，可选) |
| `academic_schedules` → `deadline_reminders` | `related_schedule_id` (BIGINT，可选) |
| `study_intentions` → `student_applications` | `intention_id` (BIGINT，可选) |
| 所有表 → `test.students` | `student_id` (BIGINT) |
